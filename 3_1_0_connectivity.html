Connectivity Interfaces#

Overview#

The DevLab Multi Hub Shield integrates multiple industry-standard communication protocols, providing comprehensive connectivity options for sensors, displays, and peripheral modules. This section details each interface and provides practical guidance for implementation.

I2C Interface#

Description

Inter-Integrated Circuit (I2C) is a synchronous, multi-master, multi-slave serial communication protocol. The shield implements I2C as the primary interface for:

  • OLED display communication

  • QWIIC/STEMMA QT connectors

  • Sensor modules

  • Expansion devices

Technical Specifications

Table 8 I2C Specifications#

Parameter

Value

Protocol

Standard (100 kHz), Fast (400 kHz)

Voltage Level

3.3V (compatible with 5V via level shifting)

Pull-up Resistors

Integrated on-board (4.7k ohms typical)

Maximum Devices

127 addressable devices

Connectors

7x QWIIC/STEMMA QT (JST 1mm)

Pin Assignment

  • SDA (Serial Data): Bidirectional data line

  • SCL (Serial Clock): Clock signal generated by master

  • VCC: Power supply (3.3V or 5V)

  • GND: Common ground

Programming Example

#include <Wire.h>

void setup() {
  Wire.begin();        // Initialize I2C as master
  Serial.begin(115200);
}

void loop() {
  // Read from I2C device at address 0x3C
  Wire.beginTransmission(0x3C);
  Wire.write(0x00);    // Register address
  Wire.endTransmission();

  Wire.requestFrom(0x3C, 1);  // Request 1 byte
  if (Wire.available()) {
    byte data = Wire.read();
    Serial.println(data, HEX);
  }
  delay(1000);
}

SPI Interface#

Description

Serial Peripheral Interface (SPI) is a high-speed, full-duplex synchronous serial communication protocol. The shield provides SPI connectivity for:

  • High-speed sensors

  • SD card modules

  • External memory

  • Display modules

Technical Specifications

Table 10 SPI Specifications#

Parameter

Value

Protocol

SPI (4-wire)

Speed

Up to 20 MHz (platform dependent)

Mode

Master only

Voltage Level

3.3V logic

Chip Selects

Multiple CS pins available

Pin Assignment

  • MOSI (Master Out Slave In): Data from master to slave

  • MISO (Master In Slave Out): Data from slave to master

  • SCK (Serial Clock): Clock signal from master

  • CS/SS (Chip Select): Device selection (active low)

Programming Example

#include <SPI.h>

#define CS_PIN 10

void setup() {
  pinMode(CS_PIN, OUTPUT);
  digitalWrite(CS_PIN, HIGH);  // Deselect

  SPI.begin();
  SPI.setClockDivider(SPI_CLOCK_DIV4);  // 4 MHz
  SPI.setDataMode(SPI_MODE0);
}

void loop() {
  digitalWrite(CS_PIN, LOW);   // Select device

  byte response = SPI.transfer(0xAA);  // Send and receive

  digitalWrite(CS_PIN, HIGH);  // Deselect
  delay(100);
}

UART Interface#

Description

Universal Asynchronous Receiver-Transmitter (UART) provides serial communication for:

  • GPS modules

  • Bluetooth modules

  • Debug console

  • Serial sensors

Technical Specifications

Table 12 UART Specifications#

Parameter

Value

Protocol

Asynchronous serial

Baud Rates

9600, 19200, 38400, 57600, 115200 bps

Data Bits

8 bits (standard)

Parity

None, Even, Odd

Stop Bits

1 or 2

Voltage Level

3.3V TTL

Pin Assignment

  • TX (Transmit): Data output from shield

  • RX (Receive): Data input to shield

  • GND: Common ground reference

Programming Example

void setup() {
  Serial.begin(115200);      // USB serial
  Serial1.begin(9600);       // Hardware UART
}

void loop() {
  // Forward data between Serial and Serial1
  if (Serial.available()) {
    Serial1.write(Serial.read());
  }

  if (Serial1.available()) {
    Serial.write(Serial1.read());
  }
}

GPIO (General Purpose Input/Output)#

Description

GPIO pins provide flexible digital input/output functionality for:

  • Button reading

  • LED control

  • Relay activation

  • Digital sensors

  • Interrupt handling

Technical Specifications

Parameter

Value

12-20 mA (platform dependent)

Programming Example

#define LED_PIN 13
#define BUTTON_PIN 5

void setup() {
  pinMode(LED_PIN, OUTPUT);
  pinMode(BUTTON_PIN, INPUT_PULLUP);

  // Attach interrupt
  attachInterrupt(digitalPinToInterrupt(BUTTON_PIN),
                  buttonPressed, FALLING);
}

void loop() {
  // Main code here
}

void buttonPressed() {
  digitalWrite(LED_PIN, !digitalRead(LED_PIN));
}

ADC (Analog to Digital Converter)#

Description

The ADC interface allows reading analog voltages from:

  • Potentiometer

  • Analog sensors

  • Voltage dividers

  • Variable resistors

Technical Specifications

Table 14 ADC Specifications#

Parameter

Value

Resolution

10-12 bits (platform dependent)

Voltage Range

0 - 3.3V (typical)

Channels

Multiple ADC inputs available

Sampling Rate

Up to 200 ksps (platform dependent)

Reference Voltage

Internal or external

Programming Example

#define POT_PIN A0

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);  // 12-bit resolution (if supported)
}

void loop() {
  int rawValue = analogRead(POT_PIN);
  float voltage = rawValue * (3.3 / 4095.0);

  Serial.print("Raw: ");
  Serial.print(rawValue);
  Serial.print(" Voltage: ");
  Serial.println(voltage);

  delay(100);
}

Interface Selection Guide#

When to Use I2C

  • Multiple devices on same bus

  • Low to moderate data rates

  • Sensor modules

  • Display communication

  • QWIIC/STEMMA QT ecosystem

When to Use SPI

  • High-speed data transfer

  • SD cards, memory chips

  • Fast displays

  • Real-time applications

  • Point-to-point communication

When to Use UART

  • GPS modules

  • Wireless modules (Bluetooth, Wi-Fi)

  • Debug output

  • Simple two-wire communication

  • Legacy serial devices

When to Use GPIO

  • Simple on/off control

  • Button inputs

  • LED outputs

  • Interrupt-based events

  • Bit-banging custom protocols

Best Practices#

I2C

  • Always check for address conflicts

  • Use I2C scanner to detect devices

  • Keep cable lengths short (<30cm for reliable operation)

  • Verify pull-up resistors are present

SPI

  • Use shortest possible wires for high-speed operation

  • Properly manage chip select signals

  • Check SPI mode compatibility

  • Consider signal integrity at high speeds

UART

  • Match baud rates exactly between devices

  • Use common ground reference

  • Consider flow control for reliable communication

  • Avoid ground loops in long cable runs

GPIO

  • Never exceed maximum current ratings

  • Use current-limiting resistors for LEDs

  • Implement proper debouncing for buttons

  • Be cautious with voltage levels (3.3V vs 5V)

ADC

  • Filter noisy signals in hardware

  • Average multiple readings for stability

  • Consider reference voltage accuracy

  • Account for input impedance
